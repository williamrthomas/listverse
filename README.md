## What is ListVerse?

ListVerse is a meta-repository that catalogs the best "awesome list" style repositories on GitHub. Rather than maintaining yet another awesome list, ListVerse provides structured, searchable data about the lists themselves — with editorial commentary, accurate metadata, and cross-references.

The focus areas are **Data, AI/ML, AI Agents, Data Visualization, Data Engineering, and Developer Tools** — the domains where awesome lists have become essential discovery tools.

Data is available in both **JSON** and **CSV** formats for easy integration into tools, scripts, and dashboards.

## Schema

Each entry in [`data/lists.json`](data/lists.json) follows this schema:

```json
{
  "id": "awesome-public-datasets",
  "name": "Awesome Public Datasets",
  "github_url": "https://github.com/awesomedata/awesome-public-datasets",
  "description": "A topic-centric list of high-quality public data sources.",
  "maintainer": "awesomedata",
  "category": "Data Sources & Datasets",
  "subcategory": "Public Datasets",
  "tags": ["datasets", "open-data", "public-data", "research"],
  "stars_approx": "291k",
  "license": "Unknown",
  "last_activity": "2026",
  "entry_count_approx": "500+",
  "format": "markdown",
  "has_contributions_guide": true,
  "editorial_notes": "One of the most comprehensive public dataset directories...",
  "related_lists": ["awesome-data", "awesome-datascience"],
  "added_date": "2026-04-05"
}
```

### Field Reference

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | URL-safe slug derived from repo name |
| `name` | string | Display name of the awesome list |
| `github_url` | string | Full GitHub URL |
| `description` | string | 1-2 sentence description |
| `maintainer` | string | GitHub username or org |
| `category` | string | Primary category |
| `subcategory` | string | Specific classification within category |
| `tags` | string[] | Topic tags for cross-referencing |
| `stars_approx` | string | Approximate GitHub stars ("k" notation) |
| `license` | string | SPDX license identifier or "Unknown" |
| `last_activity` | string | Year of most recent meaningful commit |
| `entry_count_approx` | string | Rough count of entries in the list |
| `format` | string | "markdown", "json", "csv", or "mixed" |
| `has_contributions_guide` | boolean | Whether the list accepts community contributions |
| `editorial_notes` | string | Editorial commentary on quality and scope |
| `related_lists` | string[] | IDs of related lists in this collection |
| `added_date` | string | Date added to ListVerse |
| `stars_count` | integer | Live GitHub star count (auto-refreshed monthly) |

## How to Use

### Browse the Data

- **JSON**: [`data/lists.json`](data/lists.json) — Full structured data, ideal for programmatic use
- **CSV**: [`data/lists.csv`](data/lists.csv) — Flat format for spreadsheets and data tools (arrays are pipe-delimited)

### Filter with jq

```bash
# All AI & Machine Learning lists
jq '[.[] | select(.category == "AI & Machine Learning")]' data/lists.json

# Lists with 10k+ stars
jq '[.[] | select(.stars_approx | test("^[0-9]+k$") and (rtrimstr("k") | tonumber) >= 10)]' data/lists.json

# Search by tag
jq '[.[] | select(.tags | index("llm"))]' data/lists.json

# Get editorial notes for a specific list
jq '.[] | select(.id == "awesome-llm") | .editorial_notes' data/lists.json
```

### Load in Python

```python
import json

with open("data/lists.json") as f:
    lists = json.load(f)

# Filter by category
ai_lists = [l for l in lists if l["category"] == "AI & Machine Learning"]
print(f"Found {len(ai_lists)} AI/ML lists")
```

### Load in pandas

```python
import pandas as pd

df = pd.read_csv("data/lists.csv")
print(df.groupby("category").size().sort_values(ascending=False))
```

## Contributing

Contributions are welcome! To suggest a new list:

1. Verify the repo exists and is actively maintained
2. Check it's not already included (search `data/lists.json`)
3. Open an issue with:
   - GitHub URL
   - Suggested category and subcategory
   - Brief editorial note on why it should be included
4. Or submit a PR adding the entry to `data/lists.json` (and regenerate the CSV)

### Quality Guidelines

- Lists should have clear organization and be easy to navigate
- Prefer actively maintained lists (updated within the last 2 years)
- Lists with contribution guides are preferred
- Niche lists with excellent content quality are welcome regardless of star count
